# Project N.O.M.A.D. — Synology Container Manager Deployment

[![Build ARM64 Images](https://github.com/jarvisxyz/project-nomad-synology/actions/workflows/build-arm64.yml/badge.svg)](https://github.com/jarvisxyz/project-nomad-synology/actions/workflows/build-arm64.yml)

This repository provides a ready-to-use **Synology Container Manager** deployment package for [Project N.O.M.A.D.](https://github.com/Crosstalk-Solutions/project-nomad) — a self-hosted offline resource hub designed for field operations, remote deployments, and disaster preparedness.

It also maintains a multi-architecture Docker image build pipeline (`linux/amd64` + `linux/arm64`) so that ARM64 Synology models (like the DS224+) work out of the box — the upstream project publishes x86-64 images only.

> **Upstream project:** https://github.com/Crosstalk-Solutions/project-nomad

---

## Compatibility

| Architecture | Example Models | Image to Use |
|---|---|---|
| x86-64 (Intel/AMD) | DS923+, DS1522+, DS1823xs+, RS1221+ | ✅ `ghcr.io/crosstalk-solutions/...` (upstream) or `ghcr.io/jarvisxyz/...` (this repo) |
| ARM64 (Cortex-A55) | DS224+, DS423+, DS423, DS124 | ✅ `ghcr.io/jarvisxyz/...` — built by this repo's workflow |

The `docker-compose.yml` in this repo defaults to the `ghcr.io/jarvisxyz/...` images (multi-arch). If you're on x86-64, you can swap to the upstream `ghcr.io/crosstalk-solutions/...` images — commented alternatives are provided inline.

---

## Prerequisites

- **DSM 7.2** or later
- **Container Manager** installed (Package Center → Container Manager)
- **SSH access** enabled (Control Panel → Terminal & SNMP → Enable SSH service)
- **At least 4 GB RAM** recommended (8 GB+ if running Ollama models)
- **Available ports:** 8888 (admin UI), 9999 (Dozzle logs)

> **Port note:** Synology DSM uses ports 5000 / 5001 by default. Some packages (e.g. Web Station) can also claim port 8080, so this compose file maps the admin UI to **port 8888** to avoid conflicts.

## Quick Start

### Option 1: Automated Setup (Recommended)

SSH into your Synology and run the setup script in one command:

```bash
# SSH into your Synology
ssh admin@192.168.1.x

# Download and run the setup script
curl -fsSL https://raw.githubusercontent.com/jarvisxyz/project-nomad-synology/main/setup.sh | bash

# Start the stack
cd /volume1/docker/project-nomad
docker compose up -d
```

The script will:
- Create all required directories under `/volume1/docker/project-nomad/`
- Generate secure random secrets (`APP_KEY`, `MYSQL_ROOT_PASSWORD`, `MYSQL_PASSWORD`)
- Prompt for your NAS IP address
- Write a populated `.env` file
- Download `docker-compose.yml`

---

### Option 2: Manual Setup

```bash
# 1. SSH into your Synology
ssh admin@192.168.1.x

# 2. Create directories
mkdir -p /volume1/docker/project-nomad/{storage,mysql,redis}

# 3. Copy files
cd /volume1/docker/project-nomad
curl -fsSL https://raw.githubusercontent.com/jarvisxyz/project-nomad-synology/main/docker-compose.yml -o docker-compose.yml
curl -fsSL https://raw.githubusercontent.com/jarvisxyz/project-nomad-synology/main/.env.example -o .env

# 4. Edit .env — fill in your NAS IP and generate secrets
nano .env

# To generate a secret:
#   openssl rand -hex 32

# 5. Start the stack
docker compose up -d
```

---

### Option 3: Container Manager GUI

1. Open **Container Manager** in DSM
2. Click **Project** → **Create**
3. Enter project name: `project-nomad`
4. Set path: `/volume1/docker/project-nomad`
5. Click **Create docker-compose.yml** → paste the contents of `docker-compose.yml` from this repo
6. Click **Next** → configure environment variables
7. Click **Done**

> The GUI method still requires you to create the `.env` file via SSH first, or enter each variable manually in the wizard.

## Updating

Pull the latest images and restart:

```bash
cd /volume1/docker/project-nomad
docker compose pull
docker compose up -d
```

Or use the built-in **Update** feature in the N.O.M.A.D. admin UI (Command Center → Settings → Updates).

## Troubleshooting

### Docker socket permissions

If containers fail with permission errors on `/var/run/docker.sock`, ensure your Synology user is in the `docker` group:

```bash
synogroup --add docker $USER
# Then log out and back in
```

### disk-collector `rslave` error on DSM

Some DSM kernel versions don't support `rslave` bind-mount propagation. If `nomad_disk_collector` fails to start, edit `docker-compose.yml` and change:

```yaml
- /volume1:/host:ro,rslave
```
to:
```yaml
- /volume1:/host:ro
```

Disk usage reporting may be less accurate but the rest of the stack will work normally.

### Port conflicts

Check what's using a port:
```bash
netstat -tulpn | grep :8888
```

If port 8888 is taken, edit the `ports:` mapping in `docker-compose.yml` (e.g. `"8889:8080"`) and update `URL=` in `.env` accordingly.

### ARM64 image not found / pull error

The ARM64 images are built weekly by this repo's GitHub Actions workflow. If you get a pull error:
- Check the [Actions tab](https://github.com/jarvisxyz/project-nomad-synology/actions) to confirm the latest build succeeded
- Try pulling manually: `docker pull ghcr.io/jarvisxyz/project-nomad:latest`
- See [ARM64.md](./ARM64.md) for full architecture details

### MySQL startup slow

On first start, MySQL may take 60–90 seconds to initialise. The `admin` service will wait (it has `depends_on: mysql: condition: service_healthy`). This is normal.
